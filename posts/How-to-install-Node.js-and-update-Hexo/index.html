Node.js 的安装与 Hexo 的升级

0. 前言

静态博客框架 Hexo 的 Butterfly 主题在 3.3.0 版本之后只支持 Hexo 5.0.0 及其以上版本，于是我便将 Node.js、Hexo与 Butterfly 一起更新一下，为方便后续学习与交流，便写下这篇文章！

本文不包含 Hexo 博客框架的搭建，本文只在原基础上对 Hexo 框架进行升级，要想知道如何搭建 Hexo，请在本站搜索【博客的搭建与维护】并前往查看，或直接查看本站顶置文章！

2. Node.js 的卸载

会安装也要会卸载，简单介绍一下 Node.js 的卸载。

首先，使用某某软件管家或者使用 Windows 自带的控制面板进行卸载。

然后重启电脑，或者进入任务管理器干掉所有与 Node.js 相关的进程。

再然后寻找这些文件，并删除它们（当然也有可能不存在，都是正常现象）：

最后两个路径表示全局模块的存放路径和 cache 的默认存放路径，如果更改了默认存放路径，就前往更改的路径中删除！

检查环境变量中是否有与 Node.js 相关的环境变量。

如果还没有完全卸载，在 CMD 中输入 where node 指令，找到所在位置删除目录。

最后，重启一手！卸载完事！ 🎉

3. Node.js 的升级

【无脑升级法】如果需要升级的话，可以使用国内各大厂商的 xxx 软件管家对 Node.js 进行升级。

【普通升级法】当然，如果电脑上没有 xxx 软件管家也可以前往 Node.js 官网，下载最新的长期支持版本安装包，然后双击安装包进行覆盖安装即可对当前的 Node.js 版本进行升级。

如果出现上图所示的信息，证明我们的 Hexo 是安装成功的。

但是需要注意，hexo-cli 并不是 Hexo 的版本。

如果想要知道 Hexo 的版本，我们需要前往博客对应的文件夹，在博客文件夹使用 Git 的命令行（git bash），然后输入 hexo v 查看 Hexo 的版本。
